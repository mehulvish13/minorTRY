CHAPTER 1 : INTRODUCTION

Introduction
 Fitness Exercise Predictor is a computer vision and machine learning based system that detects workout type from body pose features and provides real-time repetition counting with form feedback. The system uses webcam input to capture body landmarks, computes pose angles, and predicts exercises such as push-up and squat using a trained classification model. Along with realtime detection, the project also provides an offline prediction module and a FastAPI backend with a browser UI for testing and integration.

Problem Statement
Many users perform workouts without expert supervision, which can lead to poor posture, low-quality repetitions, and risk of injury. Existing low-cost solutions mostly provide step counting or generic guidance but do not offer pose-driven exercise recognition with immediate corrective feedback. This project addresses that gap by developing a practical system that detects exercise type, counts reps, and evaluates form in real time using a standard webcam.

Project Objectives & Goals
The main objectives of this project are:
To capture body pose landmarks from live webcam feed.
To compute robust angle-based features from pose landmarks.
To train and use a machine learning model for exercise classification.
To count repetitions based on stage transitions and movement thresholds.
To assess form quality and provide instant feedback messages.
To expose model prediction through a simple API and browser interface.
To maintain low-cost deployment using open-source tools.

Scope & Limitations
Scope of the Project
Supports real-time webcam-based detection of selected exercises.
Provides exercise prediction, repetition counting, and form scoring.
Includes offline prediction from dataset rows for testing.
Offers FastAPI endpoint and browser UI for model interaction.
Can be extended to additional exercises and user-session logging.

Limitations of the Project
Currently supports a limited set of exercises in the trained dataset.
Accuracy may decrease under poor lighting or partial body visibility.
Form rules are threshold-based and may not cover all body types equally.
No persistent production database is integrated in current version.
Performance depends on webcam quality and device processing capacity.

CHAPTER 2 : REQUIREMENT ANALYSIS AND SPECIFICATION

2.1 Functional Requirement (What the software should do)
Realtime Pose Capture: Capture frames from webcam and extract human pose landmarks.
Feature Engineering: Calculate side and key joint angles (shoulder, elbow, hip, knee, ankle) from detected landmarks.
Exercise Classification: Predict exercise class from processed feature vector using trained model.
Repetition Counter: Detect movement stages and increment rep count for valid cycles.
Form Assessment: Evaluate posture/depth/lockout/tempo and produce score with feedback.
Offline Prediction: Predict class from stored CSV row features for quick validation.
Backend API: Accept numeric feature input and return exercise + form analysis.
UI Testing Page: Provide browser-based interface for submitting sample payloads.
Model Management: Load trained model, scaler, and encoders for inference.

2.2 Non-Functional Requirements (How the software should behave)
Real-time responsiveness with smooth frame processing on standard laptop hardware.
Prediction stability through angle smoothing and frame-buffered classification.
Usability with simple commands and straightforward browser test UI.
Dependability via input validation, expected feature-size checks, and error handling.
Portability across Windows/Linux/macOS Python environments.
Maintainability through modular scripts for train, predict, realtime, and backend.
Security by validating API input and limiting runtime exposure to local deployment.

CHAPTER 3: SYSTEM DESIGN AND ARCHITECTURE

3.1 System Architecture Diagram
Components:
Frontend (Browser UI): Allows user to send feature vectors and view prediction/form response.
Realtime Interface (OpenCV Window): Displays webcam stream with pose landmarks and workout metrics.
Backend/API (FastAPI): Handles request validation, model inference, and response formatting.
ML Module (scikit-learn): Classifies exercise from scaled feature vectors.
Pose Module (MediaPipe): Detects body landmarks from frames.
Output Module: Returns/display exercise class, rep count, form status, score, and feedback.

3.2 Database Design (Entity-Relationship Diagram, Schema)
Current implementation stores training/prediction data in CSV and model artifacts in PKL files.

Proposed entities for future persistent storage:
User: Stores user profile and preferences.
Workout_Session: Stores workout start/end and summary statistics.
Exercise_Record: Stores frame/window-level predictions and angle metrics.
Form_Feedback: Stores score and tip messages per rep/session.
Device: Stores device/camera metadata.

Relationships
User -> Workout_Session (1:M): One user can have multiple workout sessions.
Workout_Session -> Exercise_Record (1:M): One session can include many prediction records.
Exercise_Record -> Form_Feedback (1:M): One record/rep can have multiple feedback messages.
Device -> Workout_Session (1:M): One device can be used for multiple sessions.

3.3 FlowChart
The system starts by initializing webcam and loading trained model artifacts.
Each frame is processed to detect pose landmarks using MediaPipe.
The system checks visibility of required joints and selects body side.
Angle-based features are computed and scaled.
The model predicts exercise class and updates prediction smoothing buffer.
Stage transitions are evaluated to count valid repetitions.
Form metrics are scored and feedback is generated.
Output is rendered in realtime window and/or returned through API.

CHAPTER 4 : IMPLEMENTATION OVERVIEW

4.1 Tools and Technologies Used
The Fitness Exercise Predictor system is implemented using technologies suitable for realtime computer vision and ML inference.

Python: Primary language for model training, inference, API, and realtime processing.
OpenCV: Webcam capture, frame processing, visualization overlays.
MediaPipe: Realtime pose landmark detection.
scikit-learn: Random Forest model training and prediction.
Pandas: CSV loading and preprocessing.
NumPy: Numerical operations on feature arrays and angles.
Joblib: Model, scaler, and encoder serialization.
FastAPI: Backend API for prediction service.
Uvicorn: ASGI server for FastAPI deployment.
Pydantic: Request validation and schema handling.
VS Code: Development, debugging, and project management.

4.2 Algorithm And Model Description
A labeled angle-feature dataset is loaded from CSV.
Categorical columns (side, label) are encoded.
Features are normalized using StandardScaler.
A Random Forest classifier is trained and evaluated using train-test split.
Trained model, scaler, and encoders are serialized as PKL artifacts.
In realtime mode, webcam frames are captured continuously.
MediaPipe detects pose landmarks and required joints are extracted.
Joint angles are calculated and transformed into model input feature vector.
Features are scaled and passed to the classifier for exercise prediction.
Prediction buffering and smoothing improve temporal stability.
Rule-based checks evaluate movement depth, lockout, alignment, and tempo.
Rep counter increments only for valid movement cycles.
Final output (exercise, reps, form status, score, feedback) is displayed.
